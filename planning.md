# Provenance Guard - Planning

## Project Overview

Provenance Guard is a web API that analyzes submitted text and estimates whether it was written by a human or generated by AI. Instead of making absolute claims, the system combines multiple detection signals into a confidence score and presents a transparency label that communicates uncertainty. Every decision is stored in an audit log, and creators can appeal classifications they disagree with.

---

# Architecture Narrative

## Submission Flow

1. A user submits text through the `POST /submit` endpoint.
2. The API validates the request to ensure text is present.
3. The submitted text is analyzed using **Detection Signal 1 (Groq LLM)**.
4. The same text is analyzed using **Detection Signal 2 (Stylometric Heuristics)**.
5. Both signal scores are combined into a single confidence score.
6. Based on the confidence score, the system generates an attribution result and transparency label.
7. The complete decision (including signal scores, confidence, and label) is written to the audit log.
8. The API returns the classification result, confidence score, transparency label, and signal information to the user.

## Appeal Flow

1. A creator submits an appeal through the `POST /appeal` endpoint.
2. The request includes the original content ID and the creator's explanation.
3. The content status is updated to **Under Review**.
4. The appeal is recorded in the audit log alongside the original classification.
5. The API returns a confirmation message indicating that the appeal has been received.

# Detection Signals

## Detection Signal 1: LLM-Based Classification (Groq)

### What it measures

The Groq-hosted Llama model analyzes the overall writing style, coherence, structure, wording, and patterns commonly associated with AI-generated content.

### Why this differs between human and AI writing

Large language models are capable of recognizing subtle stylistic and semantic patterns that frequently appear in AI-generated text, such as overly consistent phrasing or highly structured responses.

### Blind Spots

* Highly polished human writing may resemble AI writing.
* AI-generated text that has been heavily edited may appear human.
* Results depend on the quality of the prompt used for classification.

---

## Detection Signal 2: Stylometric Heuristics

### What it measures

The system computes measurable writing characteristics such as:

* Sentence length variance
* Average sentence length
* Vocabulary diversity (Type-Token Ratio)
* Punctuation density

### Why this differs between human and AI writing

Human writing typically contains greater variation in sentence length and vocabulary, while AI-generated writing often appears more uniform and consistent.

### Blind Spots

* Very short submissions provide too little data.
* Experienced writers may naturally write with consistent patterns.
* Edited AI text can closely resemble human writing.

---

# Confidence Scoring

The system combines both detection signals into one overall confidence score.

Example weighting:

* LLM Classification: **70%**
* Stylometric Heuristics: **30%**

Confidence ranges:

| Confidence Score | Interpretation        |
| ---------------- | --------------------- |
| 0.90 – 1.00      | High Confidence AI    |
| 0.70 – 0.89      | Likely AI             |
| 0.40 – 0.69      | Uncertain             |
| 0.10 – 0.39      | Likely Human          |
| 0.00 – 0.09      | High Confidence Human |

The confidence system is intentionally conservative because falsely labeling human writing as AI-generated is more harmful than failing to detect AI-generated content. When the two detection signals disagree, the confidence score moves closer to the uncertain range instead of forcing a confident decision.

---

# False Positive Scenario

Imagine a human writer submits a carefully edited essay.

* The Groq LLM predicts the writing is likely AI-generated.
* The stylometric analysis finds strong human writing characteristics.

Rather than confidently labeling the content as AI-generated, the combined confidence score falls into the uncertain range (for example, **0.58**).

The transparency label shown to readers becomes:

> "The system is uncertain about the origin of this content. Additional review may be needed."

If the creator believes the classification is incorrect, they can submit an appeal explaining why they disagree. The appeal updates the content's status to **Under Review**, and the entire process is recorded in the audit log.

# API Endpoints

## POST /submit

### Request

```json
{
  "text": "User submitted content"
}
```

### Response

```json
{
  "content_id": 1,
  "classification": "Likely AI",
  "confidence": 0.82,
  "label": "This content is likely AI-generated.",
  "signals": {
    "llm": 0.87,
    "stylometric": 0.71
  }
}
```

---

## POST /appeal

### Request

```json
{
  "content_id": 1,
  "reason": "I wrote this content myself."
}
```

### Response

```json
{
  "status": "Under Review",
  "message": "Appeal submitted successfully."
}
```

---

## GET /log

Returns the structured audit log containing all attribution decisions and appeals.

Example Response:

```json
[
  {
    "content_id": 1,
    "classification": "Likely AI",
    "confidence": 0.82
  },
  {
    "content_id": 2,
    "classification": "Uncertain",
    "confidence": 0.56
  }
]
```
